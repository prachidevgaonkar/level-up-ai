# Contributing to Level Up AI

Thank you for your interest in contributing. This framework is designed to be adapted, improved, and extended by practitioners who are running AI adoption programmes in the real world.

---

## How to Contribute

### Reporting Issues
If you find an error, ambiguity, or gap in the framework, open a GitHub Issue with:
- A clear title describing the problem
- The file and section where the issue appears
- A suggested fix or improvement (optional but helpful)

### Submitting Improvements
1. Fork the repository
2. Create a branch: `git checkout -b improve/your-description`
3. Make your changes
4. Submit a pull request with a clear description of what you changed and why

### Areas Where Contributions Are Most Valued

- **Role-specific quest variants** — Quest designs tailored to specific functions (e.g. HR, Finance, Engineering, Sales)
- **Industry adaptations** — Modified frameworks for healthcare, financial services, public sector, etc.
- **Leaderboard tooling** — Scripts or integrations that automate XP tracking and leaderboard publishing
- **LMS integrations** — Connections to Workday Learning, Cornerstone, TalentLMS, etc.
- **Translated versions** — Translations of the core framework documents into other languages
- **Season retrospectives** — Write-ups from organisations that have run the programme, with results

---

## Style Guide

- Use plain English. Avoid jargon where a simpler word works.
- Write in the present tense ("The squad lead submits..." not "The squad lead will submit...")
- Tables are preferred over long bullet lists for structured information
- All XP values must be internally consistent with the scoring model in `framework/scoring.md`

---

## Code of Conduct

Be constructive. This is a framework built for people, and contributions should be made in that spirit.
